# BB-0003: Handle race condition between browsers

If a user opens the same project on a browser in multiple tabs, the app starts behaving erratically. The changes coming from one side overrides the other. The app is never intended to work with multiple tabs, but to be graceful as much as possible, we can use a listener or a polling mechanism the get updates done externally by another tab.

So unless the user is spamming many tabs at once, occasional switches between tasks, would be graceful and the data shown would be never stale. Or changes done in two tabs can't override each other and make data loss.
