# Background
We have previously added support to host multiple projects in our data layer (indexeddb). But we never updated our UI to enable the user to create a project or switch between project. 

# To-do
You should make UI changes to add features to cover these user stories:
- When user opens the app the first time, the app should load the latest project the user was working on.
- When user opens the app the first time, and the user doesn't have any projects in the db, a new project should be created.
- User should be able to go to "Projects" screen to list and pick from a grid of projects, showing from most recently modified to least recently modified. Each project show a thumbnail of what it is along with it's name and last edited date. (Figma like projects screen)
- User should be able to open and work on different projects if he opens them in separate browser tabs. 
- User should be able to see project name he's currently working on, click on it and rename it. Initially a project is created with name Untitled.

You also make sure the current data layer is suited for the UI changes above and these additional constraints:
- Project has a created date and last modified date.
- Multiple projects can have same name, it's not unique.
- Each project keeps a thumbnail of the canvas so when many projects are present for a user, the project pane can render them without calculating and actually rendering all components, wires, etc.
